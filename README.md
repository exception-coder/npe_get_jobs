## 🚀 新功能支持（近期）

### 🔥 重大变更（优先关注）

- **AI 基础设施重构为多平台架构**：底层从单平台调用升级为可按平台动态路由（Deepseek / OpenAI / Qwen），并统一归集到 `infrastructure/ai`，后续扩展模型与平台更容易维护。
- **包结构大规模治理（DDD-lite）**：基础设施代码从历史分散目录统一迁移到顶层 `infrastructure/`，减少反向依赖与“代码难找”问题，提升长期可维护性。
- **首次使用引导升级为 AI 自然语言配置**：用户可直接用自然语言描述求职画像，系统自动解析并写入配置，显著降低首次配置门槛。

### ✅ 主要新功能

- 新增 **企业评估能力**：支持公司风险评估、历史记录分页、勾选删除/全部删除。
- 新增 **AI 匹配补充规则管理**：支持职位匹配与企业评估的可配置补充规则（前后端联动）。
- 新增 **Onboarding 配置保存 + 异步技能分析**：确认后自动落库并触发异步 AI 打招呼生成。
- 新增 **开发工具栏能力**：DevToolbar 支持本地存储管理（增删改查/清空），便于开发调试。
- 新增 **字典聚合接口**：`GET /dicts` 可一次获取所有平台字典数据。

## 🐞 BUG修复（近期）

### 🔥 重大修复（稳定性）

- 修复 **Playwright 启动 page 对象冲突**：在扩展初始化阶段增加等待，解决启动期 `Cannot find object to call __adopt__` 异常。
- 修复 **Cookie 恢复链路不稳定**：补齐 cookie 字段校验、注入顺序与重载策略，显著提升登录态恢复成功率。
- 修复 **Spring 启动多 Bean 歧义**：为 `RestClient.Builder` 提供默认主 Bean，解决 `NoUniqueBeanDefinitionException` 启动失败。
- 修复 **岗位过滤 Duplicate key 异常**：`toMap` 增加合并策略，避免重复 `encryptJobId` 导致过滤流程中断。

### ✅ 常见问题修复

- 修复 Onboarding 弹框重复弹出与 API Key 状态误判问题。
- 修复岗位搜索框清空后再次搜索 `trim()` 空指针异常。
- 修复岗位删除/重置平台参数不一致导致接口识别错误。
- 修复快速投递接口路径冲突，统一走 `/submit/{platformCode}`。
- 修复已投递岗位被“删除全部”误删除问题（改为排除投递成功/失败状态）。

## 📖 使用说明

### ⚠️ 重要：启动时设置 AI API Key

使用一切 AI 功能前，**必须在启动应用时**设置 `spring.ai.openai.api-key`。  

在 IDE（如 IntelliJ IDEA）的运行配置中，于「程序实参」里填写：

```text
--spring.ai.openai.api-key=你的密钥
```

该参数**仅用于满足启动校验**，可填写任意字符（如 `dummy` 或 `mykey`）即可。**启动后请进入「公共配置」页，在 AI 配置中填写真实的 Deepseek API Key**——本项目的 AI 功能实际使用的是 Deepseek。

![运行配置中设置 API Key](docs/images/readme/run-config-api-key.png)

---

### 第一步：配置公共信息 🔧

在开始使用之前，你需要先完成公共配置，这些配置会应用到所有投递平台：

1. 进入系统后，找到**公共配置**（访问地址一般为 `http://localhost:8081`）
2. 配置以下关键信息：
   - **黑名单过滤配置**：关键字过滤（岗位/描述）、关键字过滤（公司），智能过滤不匹配的岗位和公司
   - **候选人画像**：目标职位名称、工作年限、核心技能、职业意向，提升 AI 匹配精准度
   - **AI 配置**：选择 AI 服务提供商（如 Deepseek）、填写 API Key、开启/关闭 AI 职位匹配与智能打招呼
   - **简历与沟通配置**：管理简历和沟通相关设置

![最新版公共配置界面](docs/images/readme/common-config.png)

💡 **小提示**：公共配置只需设置一次，后续可随时调整！

---

### 第二步：配置投递平台 🎯

完成公共配置后，就可以针对具体的投递平台进行个性化配置了：

1. **选择投递平台**
   - 在左侧导航栏选择你想使用的平台（如 BOSS直聘、猎聘、51Job 等）

2. **设置检索条件**
   - **搜索关键字**：输入职位关键词（如"Java开发"、"产品经理"），可用逗号分隔多个关键词
   - **工作城市**：选择你的目标城市（支持多选）
   - **工作经验**：选择符合你的经验范围
   - **职位类型**：全职/兼职/实习
   - **薪资范围**：设置你的期望薪资区间
   - **学历要求**：选择学历要求
   - **行业筛选**：最多选择 3 个感兴趣的行业
   - **公司规模**：选择偏好的公司规模
   - **融资阶段**：选择公司融资阶段偏好

3. **保存配置**
   - 配置完成后，点击**保存配置**按钮

4. **开始投递** 🚀
   - 点击**一键投递**按钮，系统将自动开始为你筛选和投递简历
   - 你可以随时点击**手动加载任务状态**查看投递进度

### ⏱️ 为什么投递比较慢？

GetJobs 的投递流程是：**先采集岗位 → 再过滤（黑名单、AI 匹配等）→ 再逐个投递**。每个岗位都会完整走完一轮操作，且**全程模拟真人操作**，每个岗位之间会**故意延迟 3～5 秒**，因此整体会显得比较慢。

这样设计的目的：

- **为真实求职者服务**：面向个人求职者提高效率，不做批量、刷量式投递。
- **保护平台与大家**：乱投、无效投递既伤害招聘平台，也会拉低整体环境，适度、精准更可持续。
- **每天几十个有效投递就够**：质量优于数量，建议把工具当作「挂机小助手」，而不是追求极速海投。

**使用建议**：工作日下午或晚饭后开启一键投递，让程序在后台慢慢跑即可；不必追求短时间内投递大量岗位。

---

### ⚠️ 注意事项

- 建议先在单个平台测试配置是否正确，确认无误后再开启多平台投递
- 定期检查黑名单，避免错过好机会
- 合理设置投递频率，避免被平台限制
- 及时更新简历信息，保持信息的准确性
- **推荐工作日下午或晚饭后挂机投递**，每天几十个有效投递即可，重在质量而非数量
